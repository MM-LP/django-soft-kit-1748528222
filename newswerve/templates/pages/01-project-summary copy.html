Swervetracker: Project Timeline & Design Stack

Phase I – Concept Development

Early exploration of water ski performance tracking using GPS metadata, photo/video logs, and mobile sensors. Defined key goals: automate tagging, minimal input, skier/boat data sync.

Phase II– Swervetracker Consolidation

Final architecture planning. Merging slalom GPS modules into the media system, preparing for production deployment with Swiss server options, and expanding social/data sharing components.

Phase III – UI/UX + Frontend

Standardized on Tailwind-based Django templates. Added support for Creatomate overlays, mobile GIFs/stickers, and intuitive templates with gesture/voice input plans.

Phase IV – Django for Media

Created the initial media catalog backend using Django, PostgreSQL, and Tailwind CSS. Incorporated support for Sync.com cloud storage, ACDSee desktop workflows, and mobile uploads.

Phase V – Slalom GPS Tracker

Built a 10Hz GPS slalom monitoring system to track buoy passes, skier angles, and timing. This system feeds precision telemetry into the Django system for alignment with media.

Technology & Design Decisions

  • Frontend: Tailwind CSS + Django Templates (Alpine.js optional)
  • Backend: Django, Python, REST endpoints
  • Database: PostgreSQL with PostGIS support (GeoDjango)
  • Media Tools: ACDSee (admin), Sync.com (storage), Creatomate (mobile overlays)
  • Mobile Capture: Phone sensors (GPS, mic, compass) with template-based media logging
  • Hosting: Swiss-based VPS or custom Django deployment with encrypted media
